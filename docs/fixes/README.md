## 记录格式

每个修复记录应包含：

```markdown
# [问题标题]

## 问题描述
[用户可感知的现象]

## 问题根源
[技术层面的原因分析]

## 解决方案
[具体的代码修改和原理]

## 修改文件
[涉及的文件列表]

## 效果
[修复后的效果验证]
```

---

## 何时添加记录

符合以下条件的问题应记录：

1. **排查耗时 > 30 分钟** - 说明问题不简单
2. **涉及多个文件/模块** - 需要理解交互关系
3. **问题根源不直观** - 表象和原因不一致
4. **可能再次遇到** - 有复用价值
